# 移动端界面清理与适配重构方案

## 1. 现状问题分析
根据移动端（手机）截图反馈，当前界面存在以下严重问题：
- **纵向空间浪费**：顶部存在多余的浏览器/应用 Header，底部存在三层堆叠（ActionBar、Bottom Nav、系统栏），导致中间核心内容区（文件树/主视图）极度拥挤。
- **视觉层级混乱**：ActionBar 悬浮在半空，阴影与背景冲突；旧的 Session 气泡遮挡文件内容。
- **色彩对比度灾难**：在暗色背景下，文件树文字（深灰色）几乎不可见，严重影响可用性。
- **入口冗余**：底部导航的 `Sessions` 与即将在输入框引入的“蓝点”功能完全重复。

---

## 2. 布局瘦身方案 (Mobile Layout Cleanup)

### 2.1 移除顶部冗余 Header
- **操作**：在 `AppShell.tsx` 的 `isMobile` 分支中，彻底删除 `<header>` 标签及其包含的侧边栏开关和图标。
- **理由**：移动端侧边栏（Files）已由底部导航承载，Session 入口由蓝点承载，顶部空间应全量让给主内容。

### 2.2 优化底部导航 (Bottom Navigation)
- **操作**：
  - 移除 `Sessions` Tab。
  - 保留 `Files`（用于切换到文件树查看）和 `View`（用于切换回主视图内容）。
- **结构**：改为双 Tab 切换，或者在主视图内通过手势呼出文件树。

### 2.3 ActionBar 吸底停靠 (Docking)
- **操作**：
  - 针对手机端，将 `ActionBar.tsx` 的容器样式改为 `margin: 0`、`border-radius: 0`。
  - 将其固定在 `Bottom Navigation` 的正上方，作为底部工具栏的一部分，取消“悬浮”感。
  - 调整 `padding`，确保在软键盘弹起时，ActionBar 能顺滑跟随并保持最小占用。

---

## 3. 视觉体验重构 (Visual Refinement)

### 3.1 强制高对比度与暗黑模式适配
- **操作**：在 `index.css` 中引入 `@media (prefers-color-scheme: dark)`。
- **变量定义**：
  - 确保 `FileTree` 文字颜色在暗色下自动切为 `#f3f4f6` (近白色)。
  - 统一背景色层级：主视图（最底层） > 侧边栏/抽屉（中间层） > 输入框/按钮（最上层）。

### 3.2 废弃悬浮气泡，统一交互容器
- **操作**：
  - 删除 `SessionBubble.tsx` 和相关的浮动气泡代码。
  - 移动端所有的活跃交互统一收纳到 `BottomSheet` 容器中。
  - 确保 `BottomSheet` 弹出时，背景会有适当的 `backdrop-filter: blur()` 或半透明蒙层，明确交互焦点。

---

## 4. 响应式隔离策略

- **样式隔离**：优先使用 CSS 媒体查询 (`@media`) 处理宽度相关的视觉差异，避免在 JS 中过度计算 style。
- **逻辑分流**：利用 `isMobile` 钩子仅在 `AppShell` 顶层进行大块结构的条件渲染，保持子组件（如 `FileTree`, `ActionBar`）的纯粹性，通过 Props 传递 `compact` 或 `isMobile` 模式。

---

## 5. 预期效果图景
1. **顶部**：直接开始显示文件/视图内容（状态栏下方即内容）。
2. **中间**：无任何遮挡的气泡，视线通透。
3. **底部**：一个极简的 ActionBar（蓝点 + 输入框）紧贴着双 Tab 切换栏（Files/View）。
4. **拉起抽屉**：点击蓝点，一个半屏高的、高对比度的聊天面板从底部升起，盖在文件之上。
