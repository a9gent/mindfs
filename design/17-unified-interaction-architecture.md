# MindFS 2.0: 全端统一交互架构与蓝点逻辑规范

## 1. 全端统一交互架构 (Unified Cross-platform Architecture)

### 1.1 核心设计哲学
取消移动端与 PC 端完全隔离的 DOM 渲染逻辑，采用“一套代码，两套姿态”的响应式架构。通过统一的 Grid 布局实现组件的高度复用与交互逻辑的百分之百对齐。

### 1.2 布局行为准则
| 维度 | PC / 桌面端 | 移动端 (手机) |
| :--- | :--- | :--- |
| **侧边栏表现** | **固定挤压式**：占据固定宽度（左260px / 右280px），与主视图并排。 | **覆盖抽屉式**：占据屏幕 85% (左) / 75% (右) 宽度，弹出时带有半透明模糊遮罩。 |
| **初始状态** | 左右侧边栏默认 **Open**。 | 左右侧边栏默认 **Closed**。 |
| **选中交互** | 保持侧边栏状态，仅刷新主视图。 | **选中即关**：点击文件或会话项后，侧边栏立即自动滑走，将空间全量让给主视图。 |
| **控制入口** | **顶部两角图标**：侧边栏关闭时显现，展开时自动隐藏以精简 UI。 | **顶部两角图标**：常驻显示，作为进入文件树和历史的唯一入口。 |

### 1.3 容器自适应互斥 (Adaptive Mounting)
*   **原则**：活跃的 Session 具有“唯一展示性”。
*   **逻辑**：若 `mainView === activeSession`，则在主视图全屏渲染；否则，自动收纳并渲染在底部的 **AI 抽屉 (PC Overaly / Mobile BottomSheet)** 中。

## 2. 蓝点交互逻辑完善 (Refined Blue Dot Hub)

蓝点作为 AI 助手的唯一物理枢纽，承载了生命周期管理与容器切换的双重职能。

### 2.1 三态视觉状态机
1.  **初始态 (Initial)**：空心圆/灰色。表示当前无活跃绑定会话。
2.  **待绑定态 (Pending)**：实心浅蓝。用户点击了历史会话但尚未发话，处于“可反悔”阶段。
3.  **已绑定态 (Bound)**：高亮深蓝 + 发光。用户已发送消息，输入框锁定。

### 2.2 多态触发动作
*   **单击 (Click)**：
    *   **已绑定态**：弹出/收起 AI 抽屉。
    *   **全时可用逻辑**：只要主视图当前不是正在交互的活跃 Session，蓝点始终允许点击并呼出抽屉。
*   **左滑 (Slide Left)**：
    *   **触发条件**：按住蓝点向左（输入框方向）滑动位移 > 40px。
    *   **行为**：立即重置 AI 上下文，清空输入框，蓝点回退至初始态。
*   **自动弹出 (Auto-Pop)**：
    *   在“伴随模式”（主视图看文件）下发送消息，系统会强制自动拉起 AI 抽屉，确保即时看到 Agent 响应。

### 2.3 AI 抽屉 (Drawer) 视觉规范
*   **高度**：统一设定为 **75vh**，提供沉浸式阅读空间。
*   **PC 底部吸附**：抽屉下沿精准贴合 `ActionBar` 顶部（52px），消除悬浮感。
*   **不透明背景**：采用坚实不透明背景（PC: var(--sidebar-bg) / Mobile: #FFF），彻底杜绝底层文字干扰。
*   **头部压缩**：极致压缩 Handle 区域高度（12px），最大化有效交互面积。
